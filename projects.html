My Projects

Intro

I started this project at the beginning of my first semester of my second year of university. This was my first ever attempt at completing a project that wasn't directed at school work. In the completion of this project I learned a lot about managing a software project through github, CRUD operations within databases, and event driven programming which was a programming style I had never attempted in the past. I started out programming this project in an imperative way but realized over time it was much easier to program it in more of an object-oriented fashion. I consider this project finished since it accomplishes what I had set out to do but there are definitely some things I would change if I were to try this again.

Technical Background

This project was coded entirely with Python 3 but it required several external packages to make it do what I needed it to do.

  • Tkinter
    • This is the package that was used to design the graphical user interface and program functionality for the buttons
  • Beautiful Soup 4
    • This package allowed me to grab the HTML from the relevant page and parse through it to pull out the financial statement information
  • Selenium
    • I used Selenium's Chrome driver as that was the main browser I was using at the time
  • MySQL Connector
    • This package was used to create and execute SQL queries on the database that contained the financial statement information for each company
    • The database contains one table for each company that is added
  • CSV Package
    • This package allows the program to export the data within the database to a csv file in order to be represented in Excel

Main Screen

This the first screen that would be shown on any run of the program. I kept the design very simple with nothing but five buttons and a title. If I were to complete this project again I would put some more time into the design to make it more visually appealing and I would probably add a menu bar along the top with keyboard shortcuts for quick access to each function

Add Screen

This is the screen that will open if you press the "Add Company" button on the main screen. Again, it has a very simple design with a title, a brief instruction to the user, and two buttons. The user can enter the stock symbol of a company and press the add button to the right of the input box. If the info for that company already exists in the database, a pop-up window appears that allows the user to select to overwrite their existing data or cancel. If the operation is allowed to continue, Selenium web driver opens the Chrome browser, navigates to the specified page and pulls the financial statement information for the company and files it in the database. If the company cannot be found, a pop-up box will appear, prompting the user to try again. On clicking the "Return to main" button, the user will return to the main menu. If I were to do this project again, I would use the requests package as opposed to a combination of the Selenium and BeautifulSoup packages to simplify the process.

Delete Screen

This is the screen that will open if you press the "Delete Company" button on the main screen. It's design is identical to that of the Add screen with a title, brief instruction, and two buttons. The user can enter the stock symbol of a company that they no longer want the data for in their database. If a table with that name does not exist within the database, a pop-up window appears to tell the user their input was invalid. If the operation is allowed to continue, the table is simply dropped from the database and is unretrievable. On clicking the "Return to main" button, the user will return to the main menu.

Display Screen

This is the screen that appears when the user selects the "Display Database" button on the main screen. The screen has two headers, one for the company and one for the type of statement, toggle buttons for each, a table, and two other buttons. By clicking the right and left arrow buttons beside the company symbol header, the user can page through all of the companies that they have in their database. By clicking the left and right arrow buttons beside the statement title the user can page through the three financial statements pulled by the program as shown in the images above. The table is made up of a 2d array of Label elements within the Tkinter package. I could have introduced another package like Pandas and embedded one of their dataframes, but I had no desire to introduce any more packages than I already had. The user can export each table to Excel by pressing the button in the top left which creates a new Excel file, fills in the info from the table, and saves it into a folder in the same location as the program. On pressing the "Return to main" button, the user is sent back to the main menu.
